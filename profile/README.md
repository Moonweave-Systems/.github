# Moonweave Systems

Developer infrastructure for AI-assisted work: decision kernels, coding-agent skills, workflow checks, and harnesses for safer automation.

## Public Entry Points

| Project | What it is for |
| --- | --- |
| [decision-kernel](https://github.com/Moonweave-Systems/decision-kernel) | Evidence-gated decision skills, drift audits, and useful-done checks for agentic coding sessions. |
| [harness-check](https://github.com/Moonweave-Systems/harness-check) | One-shot Claude Code environment audit for hooks, plugins, MCP processes, and usage stats. |
| [decide-skill](https://github.com/Moonweave-Systems/decide-skill) | A focused skill for researching implementation choices and making reversible technical decisions. |
| [compass-skill](https://github.com/Moonweave-Systems/compass-skill) | Mid-session audit skill for drift, accumulated repo rot, and missed verification loops. |
| [anneal-skill](https://github.com/Moonweave-Systems/anneal-skill) | Measurement-first build-direction selection and iterative polish discipline for agents. |

## Scope

Moonweave Systems is for software that improves how agents build software: runtime checks, coding workflows, skill packaging, verification gates, and developer-facing automation.

Research workflow tools live in [Moonweave Research](https://github.com/Moonweave-Research). Personal utilities and experiments stay under [moonweave](https://github.com/moonweave).

## Operating Principles

- Evidence before claims: verify behavior with commands, logs, or live surfaces.
- Small, reusable surfaces: skills and tools should be understandable without reading an entire workspace.
- Agent workflows should make failure visible instead of hiding it behind retries or vague status.
- Public repos should be installable, documented, and scoped to one clear job.
